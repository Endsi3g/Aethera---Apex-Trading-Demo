## 1. Framework Technique Recommandé

Stack complet optimisé pour un déploiement kiosque en salon : **Frontend** React.js avec Tailwind CSS pour une UI responsive tactile ; **Backend temps réel** Node.js + Express + Socket.IO pour multijoueur (2-4 joueurs) ; **Base de données** MongoDB pour scénarios/sessions + Redis pour scores en temps réel ; **Graphiques** Widget TradingView embed gratuit  ; **Déploiement** Docker + Vercel/Netlify (frontend), Render/Heroku (backend), mode kiosque Edge/Chrome (--kiosk URL). Cette stack gère le temps réel via WebSockets et scale pour démos courtes.[^1][^2][^3][^4][^5][^6]

## 2. Architecture de Navigation

Navigation linéaire avec transitions fluides (5s fade) :

- **Accueil** : Boutons "Solo" / "Multijoueur" + teaser Apex.
- **Saisie Nom** : Champ texte + bouton "Jouer" (validation requise).
- **Solo** : Direct aux paramètres puis jeu.
- **Multijoueur** : Salle d'attente → Paramètres (vote) → Jeu.
- **Jeu** : Scénario → Réponse utilisateur → Explicatif Apex (10s fixe, skip impossible) → Score → Suivant.
- **Fin** : Résultats + récompense.
- États globaux : Loading spinner, bouton reset kiosque (Ctrl+Alt+Del visible en prod).


## 3. Structure de Données

```json
{
  "sessions": {
    "sessionId": {
      "players": [{ "id": "uuid", "name": "str", "score": num }],
      "roomCode": "str",
      "params": { "scenarios": 5|10|20, "difficulty": "beginner|intermediate|expert", "type": "chart|quiz" },
      "currentScenario": { "id": "str", "chartData": "TradingView config", "apexDecision": "buy|sell|hold", "explanation": { "networks": ["trend", "momentum"], "reason": "str", "risk": "str" } },
      "scores": { "playerId": num },
      "status": "waiting|playing|finished"
    }
  },
  "scenariosDB": [{ "id": "str", "difficulty": "str", "chart": "TradingView JSON", "apex": "obj" }]
}
```

Gérée via MongoDB (scénarios prédéfinis, 100+ par difficulté) et Socket.IO pour sync temps réel.[^7]

## 4. Spécifications Fonctionnelles

**Mode Solo** :

- Choix params individuels.
- 5-20 scénarios séquentiels : Graph TradingView (zoom/pan tactile) + boutons "Haussier/Baissier/Stable".
- Score : +100 précision, -50 erreur vs Apex ; comparaison live.

**Mode Multijoueur** (2-4) :

- Max 4/joueurs, auto-start à 2 ou bouton "Démarrer".
- Scores sync Socket.IO, leaderboard live.
- Même scénarios pour tous, réponses simultanées (30s timer).

```
Les deux modes : Intégration widget TradingView via `<div id="tv-chart"></div>` + `new TradingView.widget({...})`.[^8][^9]
```


## 5. Système de Salle d'Attente

- Génère `roomCode` (6 chars alphanum) + lien partageable (`/join?code=ABC123`).
- UI : Liste joueurs avatars, timer astuces (toutes 15s), boutons "Inviter" (QR code), "Commencer" (host only).
- Socket.IO rooms : `socket.join(roomCode)` ; broadcast joueurs présents.
- Responsive tactile : Swipe pour astuces.[^6]


## 6. Système de Récompense et Gamification

- Points : Précision prédiction (match Apex = 100pts), bonus streak (+20/consecutif).
- Fin partie : Leaderboard, "Félicitations [Gagnant] ! Accès bêta Apex + communauté" → Formulaire email (Mailchimp embed) pour clé beta unique (générée via backend).
- Conversion : QR code beta + lien Discord ; track via Google Analytics.
- Badges : "Discipliné" (matches "hold" Apex) pour éduquer qualité vs fréquence.


## 7. Recommandations UX/UI

- **Débutants** : Gros boutons (60px+), icônes simples (↑↓↔), tooltips hover/tap ("Haussier = prix monte").
- **Experts** : Toggle "Mode expert" (détails techniques cachés par défaut), graphiques TradingView full-featured.
- Design : Palette bleu/vert (finance trust), animations Lottie légères, full-screen responsive (media queries 1024x768+), feedback haptic/tactile (vibrations CSS/JS).[^10]
- Accessibilité : Contraste AA, voix-off labels, navigation clavier.


## 8. Logique Écrans Explicatifs Apex (10s)

Timer fixe `setTimeout(10000)` non-skippable :

```
[Graphique + Verdict Apex : "HOLD"]
Réseaux activés : Tendance (baissière), Momentum faible, Structure support.
Pourquoi : Discipline > fréquence ; volatilité haute = risque asymétrique.
Avantage : Protège capital (90% non-action Apex). Score : +50 discipline.
[Animation : Réseaux allumés → Shield capital]
```

Contenu statique par scénario (DB), philosophie : "Qualité trades, ne pas agir = force" ; multi-réseaux icones (trend/momentum/structure).[^11]

## 9. Système de Vote et Paramétrage

- Salle attente : Chips "5/10/20 scénarios", "Débutant/Inter/Expert" ; vote Socket.IO (majorité, tie=host).
- Timer vote 30s, fallback solo choix.
- Broadcast : `io.to(room).emit('voteUpdate', {difficulty: 'beginner', votes: {beginner:2}})`.


## 10. Astuces Trading Débutants

- Contenu (10 astuces rotatives) : "Tendance = ami trader (ne combattez pas)", "Volatilité haute ? Attendez Apex !", "Hold souvent = capital safe".
- Fréquence : Auto-rotate 15s en attente ; format carrousel swipe tactile, icône bulle pop-up.
- Pédagogique : Lien philosophie Apex, crédible experts (réfs concepts pros).[^12]


## 11. Stratégie Calibrage Difficulté

| Difficulté | Scénarios | Graphiques | Questions | Apex Discipline |
| :-- | :-- | :-- | :-- | :-- |
| Débutant | Clairs, tendances obv. | Zoom simple, 1H | Basiques (hausse?) | 70% hold |
| Inter | Moy. volatilité | Multi-timeframe | Momentum check | 50% hold |
| Expert | Complexes, news impact | Full TradingView tools | Risque asym. | 30% hold, multi-réseaux |

Adaptatif : Solo=choix fixe ; Multi=vote ; Mix audiences via astuces overlay débutants. Scénarios pool par niveau, random sans repeat.[^13]
